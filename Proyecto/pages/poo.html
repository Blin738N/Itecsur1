¿Qué es la POO?

La Programación Orientada a Objetos (POO) es un paradigma de programación que organiza el software en torno a "objetos" que contienen datos y comportamientos. Esto facilita la reutilización del código, la modularidad y el mantenimiento.

Clases y Objetos

Una clase es una plantilla para crear objetos. Define propiedades (atributos) y comportamientos (métodos) que los objetos de esa clase tendrán. Un objeto es una instancia de una clase.

class Persona {
    String nombre;
    int edad;

    // Constructor
    public Persona(String nombre, int edad) {
        this.nombre = nombre;
        this.edad = edad;
    }

    // Método
    public void saludar() {
        System.out.println("Hola, mi nombre es " + this.nombre);
    }
}

En el ejemplo anterior, se define una clase Persona con dos atributos (nombre y edad), un constructor para inicializar los atributos y un método saludar.

Pilares de la POO

La POO se basa en cuatro pilares fundamentales:

  • Abstracción: Simplificar objetos complejos modelando solo los aspectos relevantes.
  • Encapsulamiento: Ocultar los detalles internos de un objeto y exponer solo una interfaz.
  • Herencia: Crear nuevas clases a partir de clases existentes, reutilizando código.
  • Polimorfismo: Permitir que objetos de diferentes clases respondan al mismo mensaje de manera diferente.

Ejemplo de Herencia

La herencia permite crear una nueva clase a partir de una clase existente. Por ejemplo:

class Estudiante extends Persona {
    String carrera;

    // Constructor
    public Estudiante(String nombre, int edad, String carrera) {
        super(nombre, edad); // Llama al constructor de la clase base
        this.carrera = carrera;
    }

    // Método
    public void estudiar() {
        System.out.println(this.nombre + " está estudiando " + this.carrera);
    }
}

En este ejemplo, la clase Estudiante hereda de la clase Persona y agrega un nuevo atributo (carrera) y un nuevo método (estudiar).

Ventajas de la POO

  • Reutilización de código: Las clases y objetos permiten reutilizar código de manera eficiente.
  • Modularidad: El código se organiza en módulos independientes y fáciles de mantener.
  • Escalabilidad: Es más fácil extender y modificar el código sin afectar otras partes del sistema.
  • Mantenimiento: La estructura de la POO facilita la detección y corrección de errores.
